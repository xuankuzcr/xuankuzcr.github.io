研究作品路线图

这个页面回答三个问题：我做过什么、当前重点是什么、下一步怎么走。

一句话目标

把 LIVO 从“实验室可运行”推进到“真实场景可部署、可维护、可复现”，并延展到 UMI / 具身学习闭环。

项目状态（卡片版）

FAST-LIVO1

已完成

方法与工程基线，打通 LiDAR × Vision × IMU 融合主链路。

FAST-LIVO2

已完成

主力系统版本，强调稳、快与工程可用性。

FAST-Calib

已完成

快速标定工具链，降低部署门槛。

Global-LVBA

已完成

全局一致后端优化，补强长程收敛。

LIV-Handheld

已完成

采集与同步平台，提升实验复现效率。

CamVox

已完成

低成本路线参考，平衡成本与性能。

LiDAR-Visual-RTK

已完成

补全全局绝对参考，增强跨场景重访稳定性。

LIV-GaussMap

进行中

融合几何系统向高保真表示能力延展。

UMI 主线

规划中

连接示教采集、数据质检、策略训练与评测闭环。

研究主线（Engineering + Research）

Phase A · 前端融合跑通

FAST-LIVO1/2：建立稳定实时估计主线，保障高频可用。

Phase B · 部署成本降低

FAST-Calib + LIV-Handheld：降低标定与采集门槛，提升复现效率。

Phase C · 全局一致性增强

Global-LVBA + LiDAR-Visual-RTK：补强长程收敛，服务大尺度重访。

Phase D · 表示与任务扩展

LIV-GaussMap + UMI：让地图不仅用于定位，也服务示教与操作学习。

我关心的核心问题

  1. 如何在真实场景中同时做到精度、鲁棒性、实时性？

  2. 如何让系统具备可部署、可维护、可复制的工程品质？

  3. 如何在低成本传感器组合下逼近高端系统效果（low-cost lifelong SLAM）？

  4. 如何把几何融合系统与 UMI / VLA / Diffusion Policy 打通？
